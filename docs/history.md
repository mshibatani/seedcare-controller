# ノンブロッキング設計への改修 (2026-03-02)

## 問題
ESP32が時々停止する。原因はloop()内の複数のブロッキング処理。

## 修正内容

### 停止原因と対策

| 箇所 | 旧 | 新 |
|------|-----|-----|
| WiFi再接続 | whileループ最大180秒 | 状態マシン、15秒タイムアウト |
| 温度取得 | `requestTemperatures()` ブロック | `setWaitForConversion(false)` + 2秒タイムアウト |
| MQTT再接続 | 20回×3秒delay=60秒+ | 3回試行、10秒間隔のノンブロッキング |
| リレー制御 | 逆転delay(3000)×2 | 逆転操作削除、GPIO初期化で確実に |
| loop() | `delay(10000)` | `millis()` ベースタイマー |

### 追加機能

- **ウォッチドッグタイマー**: 30秒のWDT。万が一ハングしても自動リセット
- **チェックポイントログ**: 各処理ブロックの前後にチェックポイント記録
  - RTC memoryに最終チェックポイント保持（ソフトリブートで生存）
  - SPIFFSにクラッシュ履歴保存（電源断でも生存、最大100エントリ）
- **`/logs` エンドポイント**: ブラウザからクラッシュ履歴を確認可能
- **クレデンシャル外部化**: `include/config.h` に分離（.gitignore済み）

## 不採用: PlatformIO Remote
PlatformIO Remoteは非推奨・開発終了のため不採用。既存のOTA（AsyncElegantOTA）を継続利用。
